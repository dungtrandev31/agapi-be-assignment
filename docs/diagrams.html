3. Purchase Flow — Happy Path

sequenceDiagram
    participant C as Client
    participant RL as RateLimitFilter
    participant SVC as FlashSaleService
    participant R as Redis
    participant DB as MySQL
    participant K as Kafka
    participant INV as InventorySyncConsumer

    C->>RL: POST /flash-sales/{itemId}/purchase
    RL->>RL: Check IP rate (100 req/min)

    rect rgb(235, 248, 235)
        Note over SVC,DB: @Transactional - Sync (fast ~10ms)
        SVC->>R: DECR flash_sale_stock:{itemId}
        R-->>SVC: remaining >= 0

        SVC->>DB: UPDATE sold_quantity +1 (WHERE sold < max)
        DB-->>SVC: rows = 1

        SVC->>DB: UPDATE balance -= amount (WHERE balance >= amount)
        DB-->>SVC: rows = 1

        SVC->>DB: INSERT order (status = PENDING)
        DB-->>SVC: OK
    end

    SVC-->>C: 200 {orderNo, status: PENDING}
    Note over SVC,K: @TransactionalEventListener(AFTER_COMMIT)
    SVC->>K: STOCK_DEDUCTED event (with saga context)

    rect rgb(230, 245, 255)
        Note over K,INV: Async - Kafka Consumer (any pod)
        K->>INV: Consume event
        INV->>DB: UPDATE totalStock -1 (WHERE stock >= 1)
        DB-->>INV: rows = 1
        INV->>DB: UPDATE order status = SUCCESS
        INV->>DB: INSERT inventory_event (PROCESSED)
    end

    Note over INV: Saga Complete

3b. Purchase Flow — Compensation (stock hết / lỗi)

sequenceDiagram
    participant C as Client
    participant SVC as FlashSaleService
    participant R as Redis
    participant DB as MySQL
    participant K as Kafka
    participant INV as InventorySyncConsumer

    C->>SVC: POST /purchase
    Note over SVC,DB: ... same sync steps ...
    SVC-->>C: 200 {orderNo, status: PENDING}
    SVC->>K: STOCK_DEDUCTED event

    rect rgb(255, 235, 235)
        Note over K,INV: Async - Kafka Consumer
        K->>INV: Consume event
        INV->>DB: UPDATE totalStock -1 (WHERE stock >= 1)
        DB-->>INV: rows = 0 (het stock!)

        Note over INV: Trigger Saga Compensation

        INV->>DB: DELETE order (free unique constraint)
        INV->>DB: UPDATE balance += amount (restore)
        INV->>DB: UPDATE sold_quantity -1 (restore)
        INV->>DB: INSERT inventory_event (COMPENSATED)
        INV->>R: INCR flash_sale_stock:{itemId} (restore)
    end

    Note over INV: Saga Compensated - all restored

erDiagram
    users {
        BIGINT id PK
        VARCHAR email UK
        VARCHAR phone UK
        VARCHAR password_hash
        DECIMAL balance
        BOOLEAN email_verified
        BOOLEAN phone_verified
        VARCHAR role
        VARCHAR status
        DATETIME created_at
        DATETIME updated_at
    }

    otp_verifications {
        BIGINT id PK
        VARCHAR identifier
        VARCHAR otp_code
        VARCHAR purpose
        BOOLEAN verified
        DATETIME expires_at
        DATETIME created_at
    }

    products {
        BIGINT id PK
        VARCHAR name
        TEXT description
        DECIMAL original_price
        INT total_stock
        VARCHAR status
        DATETIME created_at
        DATETIME updated_at
    }

    flash_sale_slots {
        BIGINT id PK
        DATE sale_date
        TIME start_time
        TIME end_time
        VARCHAR status
        DATETIME created_at
    }

    flash_sale_items {
        BIGINT id PK
        BIGINT slot_id
        BIGINT product_id

        INT sold_quantity
